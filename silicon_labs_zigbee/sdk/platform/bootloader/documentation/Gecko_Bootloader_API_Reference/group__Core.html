Macro Definition Documentation

#define BTL_APP_SPACE_SIZE

Size of the area set aside for the appliction.

During firmware upgrade, the bootloader will enforce that the new application image is within the boundaries set by the start of the application and the start + the value of this define. If using an NVM system, the size of this define can be reduced in order to protect the NVM system from being overwritten by an attempted application upgrade using a too large application image.
